Верификация email

Проверка базы
перед рассылкой

Каждый bounce — удар по репутации IP. Верификация перед отправкой снижает bounce rate до <1% и защищает домен от блокировки.

⚠ Почему это критично
Bounce rate >2% → провайдеры (Gmail, Mail.ru) начинают понижать репутацию. >5% → письма уходят в спам. >10% → IP попадает в blacklist. Верификация — обязательный этап перед любой рассылкой.

Прогрев и доставляемость

Прогрев домена и
тестирование inbox

Новый домен и IP без прогрева = спам. Инструменты прогрева имитируют живую переписку и повышают репутацию отправителя.

💡 Как работает прогрев
Сервис отправляет письма с вашего домена на свой пул ящиков, открывает их, отвечает, вынимает из спама. Gmail и Mail.ru видят "живую" переписку → повышают репутацию. Процесс занимает 2–4 недели.

Для email-рассылок хостинг выбирается иначе чем для сайта. Главное — отношение к абузам и репутация IP.

⚠ Важно для массовых рассылок
Hetzner, OVHcloud, DigitalOcean и большинство EU/US хостеров блокируют аккаунты при жалобах. Для серьёзных объёмов нужны абузоустойчивые провайдеры. Для чистых opt-in баз — подойдут обычные хостеры.

Не рекомендуем для рассылок

Строгая политика к email. Блокируют быстро, порт 25 закрыт по умолчанию.

Хостинг
Причина
DigitalOcean
Блокируют порт 25. Аккаунты с рассылками отключают без предупреждения.
Vultr
Порт 25 заблокирован. Нужна заявка на разблокировку. При жалобах — быстрая блокировка.
OVHcloud
Реагируют на абузы. Для транзакционных писем подходит, для массовых рассылок — риск.
AWS / EC2
Категорически не подходит. Блокируют мгновенно. Используйте только SES для транзакционных.
Google Cloud
Порт 25 заблокирован навсегда. Нельзя разблокировать даже по запросу.
Microsoft Azure
Порт 25 заблокирован. Строгая политика. Только через Azure Communication Services.
Cloudflare Workers
Не поддерживает SMTP вообще. Только через email API сторонних сервисов.
✓ Итоговые рекомендации MAILSTACK
Легитимные opt-in рассылки → Hetzner (EU) или Timeweb (РФ). Чистые IP, лучшая репутация.
Большие объёмы, нужна лояльность → DataClub, AlexHost, BuyVM. IP потребует прогрева.
Азиатская аудитория → Shinjiru (MY), AboveVPS (HK), VPSdime (SG), G-Core Labs (SG/JP).
Латинская Америка → HostDime (BR/MX/CO), Kamatera (BR), Brillant.io (BR/AR).
РФ аудитория + оплата рублями → Timeweb, Selectel, Beget, REG.RU.
Максимум RAM за минимум денег → Contabo или Netcup.
Категорически избегать → DigitalOcean, Vultr, AWS, GCP, Azure для рассылок.
